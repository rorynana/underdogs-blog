# Skill: Draft Post

## Task
분석 리포트와 스타일 가이드를 기반으로 블로그 MDX 포스트 초안을 작성한다.
post-writer 에이전트가 이 스킬을 참조하여 실행한다.

## Input
1. solution-explorer의 분석 리포트 (7개 섹션)
2. `sot/writing-style-guide.md` (톤앤매너 규칙)
3. 사용자 컨펌된 포스트 방향/강조점 (체크포인트 1 결과)

## 글쓰기 규칙

### 블로그다운 글쓰기 원칙
이 글은 기술 문서가 아니다. 읽히는 블로그 글을 쓴다.

1. **배경 스토리로 시작**
   - "왜 이걸 만들게 됐는지"부터 시작
   - 기존 업무 방식의 불편함, 체감한 문제
   - 독자가 공감할 수 있는 상황 묘사

2. **시행착오를 솔직하게**
   - 처음에 이렇게 했다가 안 돼서 바꾼 이야기
   - "삽질"한 경험, 디버깅 과정
   - MEMORY.md의 Key Learnings에서 소재 발굴

3. **감정을 넣어서**
   - 문제 해결했을 때의 쾌감
   - 막혔을 때의 답답함
   - 결과를 보고 느낀 뿌듯함
   - writing-style-guide의 감정 표현 패턴 참조

4. **기술적 결정에 이유를 붙여서**
   - "React를 선택한 이유"가 아니라 "React를 선택할 수밖에 없었던 이유"
   - 대안과 비교, 트레이드오프 설명

5. **적절한 사족/TMI**
   - 괄호 코멘트로 곁들이는 개인적 감상
   - writing-style-guide의 유머 패턴 참조

## Checkpoint
**★ 체크포인트 2**: 초안을 사용자에게 제시.
- 전체 MDX 내용을 보여주고 피드백 요청
- 수정 요청 시 반영 후 재제시
- 사용자 "확정" 후 다음 스킬(curate-media)로 진행

## Exception Handling
- 동일 slug 기존 파일 존재 시: stub("Coming soon.")이면 덮어쓰기, 실제 콘텐츠면 사용자 확인
- 분석 리포트 데이터 부족 시: 해당 섹션 축소, 다른 섹션에서 보완
