## 채널 하나 보는 데 15분

인플루언서 마케팅을 해본 사람이라면 이 루틴이 익숙할 것이다.

유튜브 채널에 들어간다. 구독자 수를 확인한다. 최근 영상을 하나하나 클릭한다. 조회수를 본다. 댓글 분위기를 읽는다. 쇼츠와 롱폼 영상의 성과가 다르니까 따로 봐야 한다. 그걸 엑셀에 정리한다. 다음 채널로 넘어간다.

채널 하나에 15분에서 20분. 하루에 10개 채널만 봐도 반나절이 날아간다.

문제는 이렇게 시간을 쏟아도 결국 "감"으로 결정한다는 거다. 구독자 50만인데 최근 영상 조회수가 3천인 채널과, 구독자 5만인데 영상마다 2만 조회를 찍는 채널. 숫자만 보면 후자가 압도적으로 좋은 건데, 구독자 수라는 허영 지표에 눈이 멀면 전자를 고른다.

이걸 왜 아직도 사람이 하고 있을까.


## 채널 방문 3초, 점수로 보여주자

<Highlight>그래서 만들었다. TubeScout.</Highlight>

유튜브 채널 페이지에 들어가는 순간, 3초 안에 해당 채널의 협업 적합성을 점수로 보여주는 크롬 확장 프로그램이다. Videos 탭이면 롱폼 점수, Shorts 탭이면 쇼츠 점수가 자동으로 전환된다.

핵심은 "구독자 수"가 아니라 <strong>실제 성과 지표</strong>로 채널을 평가한다는 것이다.

경쟁 서비스가 없는 건 아니다. SocialBlade, Noxinfluencer, Tagger 같은 SaaS가 있다. 그런데 이것들은 전부 "별도 사이트에 접속해서 채널명을 검색하는" 방식이다. 채널을 발견한 순간 바로 점수를 볼 수 없다. 탭 전환이 필요하고, 검색이 필요하고, 로딩이 필요하다.

TubeScout은 <strong>발견 순간의 즉시성</strong>에 집중했다. 유튜브에서 눈에 띄는 채널을 발견하면, 이미 점수가 떠 있다.


## 점수를 만드는 3가지 지표

여기서 가장 중요한 질문. "그래서 뭘 기준으로 점수를 매기는데?"

인플루언서 마케팅을 직접 해본 경험에서 출발했다. 채널을 평가할 때 진짜 중요한 건 세 가지다.

![Scoring Breakdown](/images/posts/creator-scoring-algorithm/scoring-breakdown.svg)

#### 1) VPS, 구독자 대비 조회율 (가중치 50%)

Views Per Subscriber. 구독자 1,000명당 영상 하나가 몇 조회를 찍는지.

구독자 수가 많아도 영상 조회수가 바닥이면 "죽은 채널"이다. 반대로 구독자가 적어도 영상마다 높은 조회율을 기록하면 활성 팬덤이 있다는 뜻이다. 이 지표 하나만으로도 채널의 실질적 영향력을 80%는 파악할 수 있다.

```
medianViews / max(1000, subscribers) * 1000
```

최근 5개 영상의 중앙값(median)을 쓴다. 평균이 아니라 중앙값인 이유는, 바이럴 영상 하나가 전체 평균을 왜곡하는 걸 방지하기 위해서다. 이상치 제거(상하위 10% 컷)도 들어간다.

#### 2) Cadence, 업로드 주기 안정성 (가중치 30%)

꾸준히 올리는 채널인지, 간헐적으로 올리는 채널인지.

업로드 주기가 일정하다는 건 채널 운영에 진심이라는 뜻이다. 마케터 입장에서는 "이 채널이 협업 컨텐츠를 제때 올려줄 수 있는가"를 판단하는 핵심 지표다.

```
CV(변동계수) = 표준편차 / 평균 업로드 간격
score = max(0, 100 * (1 - CV / 0.5))
```

변동계수(CV)가 0이면 완벽하게 규칙적인 업로드, 0.5 이상이면 0점. 간단하지만 효과적이다.

#### 3) Momentum, 성장 추세 (가중치 20%)

상승세인지 하락세인지.

같은 점수라도 최근 성과가 올라가고 있는 채널과 내려가고 있는 채널은 전혀 다르다. "지금 뜨고 있는 채널"을 캐치하려면 이 지표가 필수다.

```
ratio = avg(최근 3개 조회수) / avg(이전 5개 조회수)
score = min(100, max(0, (ratio - 0.8) * 250))
```

ratio가 1.0이면 현상 유지, 1.2 이상이면 확실한 상승세, 0.8 이하면 하락 추세다.

이 세 가지를 가중 합산하면 0~100점 사이의 총점이 나온다. 유효한 지표만으로 가중치를 재분배하는 로직도 넣었다. 좋아요가 비공개인 채널이라든지, 데이터가 부족한 경우에도 점수가 깨지지 않는다.


## 크롬 확장은 어떻게 생겼나

기술적으로는 Chrome Extension Manifest V3 기반이다. 구조는 이렇다.

1) <strong>Content Script</strong>가 유튜브 채널 페이지를 감지한다. URL 패턴(`/@`, `/channel/`, `/c/`)을 매칭하고, 현재 탭이 Videos인지 Shorts인지 구분한다.

2) Service Worker(백그라운드)에 분석 요청을 보낸다.

3) <strong>Service Worker</strong>가 YouTube Data API를 호출한다. 딱 3번이다.
   - `channels.list` : 채널 메타 + uploads 플레이리스트 ID (쿼터 1)
   - `playlistItems.list` : 최근 영상 ID 15개 (쿼터 1)
   - `videos.list` : 조회수, 좋아요, 댓글, 길이, 게시일 (쿼터 1)

4) duration(영상 길이)으로 Shorts(&lt;60초)와 Videos(60초 이상)를 분리한다.

5) 스코어링 엔진이 점수를 계산하고, Content Script로 결과를 보낸다.

6) <strong>UI가 카드로 렌더링</strong>된다. 스켈레톤 로딩이 먼저 뜨고, 데이터가 도착하면 페이드인으로 전환된다.

API 호출 최적화에 꽤 신경 썼다. YouTube Data API는 하루 쿼터가 정해져 있는데, `search.list`(쿼터 100)를 피하고 `playlistItems.list`(쿼터 1)로 대체하는 방식이 핵심이다. 채널 하나 분석에 쿼터 3만 소모한다.

캐시도 2단계로 구현했다. 메모리 캐시(빠름)와 로컬 스토리지(영속). TTL 기반으로 5분간 같은 채널을 다시 분석하지 않는다. 최대 50개까지 LRU로 관리한다.


## 하루 만에 만들었다

사실 이 프로젝트의 가장 인상적인 부분은 개발 기간이다. PRD 작성부터 크롬 웹스토어 제출까지, 하루다.

물론 혼자 한 건 아니다. Gemini(Antigravity)가 프론트엔드 담당이었다. Content Script의 UI 렌더링, 스켈레톤 애니메이션, 다크모드 대응 같은 건 전부 Gemini가 쳐냈다. 나는 스코어링 로직 설계와 API 연동 구조를 잡았고, 나머지는 AI 에이전트한테 시켰다.

## 왜 이 3가지 지표인가

지표 설계에서 가장 오래 고민한 부분이 있다. "왜 좋아요율이나 댓글 참여율은 핵심 지표에서 빠졌는가?"

처음에는 5가지 지표를 다 넣으려고 했다. VPS, Cadence, Momentum, 좋아요율, 댓글 참여율. 그런데 실제로 데이터를 돌려보니 문제가 보였다.

첫째, 좋아요 수를 비공개로 설정한 채널이 꽤 많다. YouTube가 좋아요 수 비공개 옵션을 제공한 이후로, 특히 기업 채널에서 이걸 끄는 경우가 늘었다. 핵심 지표에 넣으면 이런 채널에서 점수가 깨진다.

둘째, 댓글 참여율은 장르 편차가 너무 크다. 뷰티/먹방 채널은 댓글이 활발한데, 테크/교육 채널은 원래 댓글이 적다. 장르 보정 없이 일률적으로 적용하면 편향이 생긴다.

그래서 MVP에서는 <strong>채널 장르와 무관하게 공정한 3가지 지표</strong>만 남겼다. 좋아요율과 댓글 참여율은 v2에서 장르별 보정 로직과 함께 추가할 계획이다.

가중치도 의도적이다. VPS가 50%로 가장 높은 이유는, 결국 마케터가 인플루언서에게 기대하는 건 <strong>노출</strong>이기 때문이다. 아무리 꾸준히 올리고(Cadence) 성장세가 좋아도(Momentum), 영상 하나하나의 조회수가 구독자 대비 너무 낮으면 협업 효과가 없다.


## Shorts vs Videos, 왜 분리했나

인플루언서 마케팅을 해본 사람이라면 안다. 같은 채널이라도 쇼츠와 롱폼의 성과가 완전히 다르다.

쇼츠에서 100만 조회를 찍는 채널이 롱폼에서는 5천 조회도 못 나오는 경우가 흔하다. 반대로 롱폼에서 충성 구독자층이 탄탄한 채널이 쇼츠에서는 존재감이 없기도 하다.

TubeScout은 YouTube Data API의 `contentDetails.duration`(ISO 8601 형식)을 파싱해서, 60초 미만이면 Shorts, 60초 이상이면 Videos로 자동 분류한다. 현재 URL이 `/shorts` 탭이면 쇼츠 점수만, `/videos` 탭이면 롱폼 점수만 보여준다.

탭 전환도 자동 감지한다. YouTube가 SPA(Single Page Application) 방식이라 URL이 바뀌어도 페이지가 새로 로드되지 않는데, MutationObserver로 URL 변경을 감시하고 1초 딜레이 후 재분석을 트리거한다.

이 "포맷별 분리"가 기존 분석 도구들과의 가장 큰 차이점이다. SocialBlade나 Noxinfluencer는 채널 전체 통계를 보여주지, 쇼츠와 롱폼을 나눠서 점수를 매기지 않는다.


## 보안과 쿼터

크롬 확장에서 API 키를 다루는 건 꽤 민감한 문제다.

MVP 단계에서는 실용 우선으로 API Key를 확장 내부에 보관했다. 테스트 용도라 수용 가능한 리스크였다. 하지만 실서비스로 가려면 반드시 서버리스 프록시(Cloudflare Workers나 Vercel Edge)로 키를 숨겨야 한다. 확장에서는 프록시 URL만 호출하고, 실제 API 키는 서버 측에서 관리하는 구조다.

YouTube Data API의 일일 쿼터는 10,000 유닛이다. TubeScout은 채널 하나에 3유닛을 쓰니까, 하루에 약 3,300개 채널을 분석할 수 있다. 개인 사용이라면 충분하고, 팀 단위로 쓰려면 프록시에서 키 풀링(여러 API 키 로테이션)이 필요하다.

에러 처리도 꼼꼼하게 넣었다. 쿼터 초과(403), 네트워크 타임아웃(10초), 레이트 리밋(429), 서버 에러(5xx). 각각 다른 에러 메시지를 보여주고, 지수 백오프로 자동 재시도한다. 최대 4번까지.


## 채널 평가, 15분에서 3초로

실제로 써보면 체감이 크다.

유튜브에서 채널을 돌아다니다가 "이 채널 괜찮은데?" 싶으면, 이미 우상단에 점수가 떠 있다. 70점 이상이면 한번 더 들여다보고, 50점 이하면 넘긴다. 찜하기 버튼을 누르면 chrome.storage에 저장되고, 팝업에서 찜한 채널 목록을 한눈에 볼 수 있다.

채널 평가에 15분 쓰던 게 3초로 줄었다. 하루에 50개 채널도 거뜬하다.

물론 한계도 있다. 점수가 높다고 무조건 좋은 협업 파트너는 아니다. 채널의 주제, 타겟 오디언스, 브랜드 핏 같은 건 알고리즘이 판단할 수 없는 영역이다. TubeScout은 "이 채널이 숫자적으로 건강한가"를 빠르게 필터링하는 도구이지, 최종 의사결정을 대체하는 건 아니다.

그래도 15분짜리 루틴을 3초로 줄인 것만으로도 충분히 가치가 있다고 생각한다. 필터링이 빨라지면, 진짜 중요한 판단(브랜드 핏, 컨텐츠 퀄리티)에 더 많은 시간을 쓸 수 있으니까.


## 마케터가 크롬 확장을 만든다는 것

비개발자 마케터가 크롬 확장 프로그램을 만들었다고 하면 대부분 의아해한다. "그게 가능해?"

가능하다. 정확히는, 내가 코드를 한 줄 한 줄 다 짠 건 아니다. 설계를 했다. PRD를 썼고, TRD(Technical Requirements Document)를 정의했고, 스코어링 공식을 만들었다. 그리고 AI 에이전트한테 구현을 시켰다.

중요한 건 "뭘 만들어야 하는지"를 정확히 아는 것이다. 채널당 15분이 걸리는 병목을 발견한 것도, 구독자 수가 허영 지표라는 걸 깨달은 것도, VPS라는 지표를 설계한 것도 전부 마케팅 실무 경험에서 나왔다.
