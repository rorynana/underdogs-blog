## Problem

Marketing data was fragmented across platforms. 광고 데이터는 Meta Ads Manager에, 매출 데이터는 내부 ERP에, 채널별 성과는 각각의 플랫폼에 흩어져 있었다.

의사결정을 위해 매번 여러 플랫폼을 오가며 데이터를 수동으로 취합해야 했고, 이 과정에서 시간이 낭비되고 실시간성이 떨어졌다.

## System

데이터 흐름을 다음과 같이 설계했다:

- **Ad data** (Meta, Google Ads) → API 수집
- **Sales data** (ERP) → 자동 추출
- **Channel data** (YouTube, Instagram) → API 수집
- → **Integrated Dashboard**

## Implementation

```python
# 데이터 수집 자동화 예시
def collect_ad_data():
    meta_data = fetch_meta_ads_api()
    google_data = fetch_google_ads_api()
    return merge_data(meta_data, google_data)
```

Google Sheets를 중간 데이터 레이어로 활용하여 비개발자도 데이터를 확인할 수 있도록 했다.

## Result

- 마케팅 의사결정 속도 향상
- 실시간 모니터링 가능
- 수동 데이터 취합 시간 제거

## Insight

데이터 통합의 핵심은 완벽한 자동화가 아니라, 의사결정에 필요한 데이터를 한 곳에서 볼 수 있게 하는 것이다.
